## Overview

This repository contains the full experimental pipeline for the thesis, which
investigates whether mBERT and XLM-RoBERTa have implicitly acquired sensitivity
to the Equivalence Constraint (Poplack, 1980), the grammatical rule governing
where language switches may occur in intrasentential code-switching, without
any explicit training on code-switching grammar.

The experiment uses a minimal pair paradigm on naturally occurring Spanish-English
conversations from the Bangor Miami Corpus. For each valid code-switched utterance,
one or more invalid counterparts are generated by moving the switch point to an
EC-violating position and translating the affected tokens using neural machine
translation. Pseudo-perplexity is then computed for each sentence under two scoring
methods, and a linear mixed effects model is used to test whether sentence validity
predicts perplexity after controlling for speaker and conversation-level variation.

## Repository structure

```
preprocess_miami.py      Parse raw CHAT files and produce enriched CSV
build_pairs.py           Generate minimal pairs with MarianMT translation
score_perplexity.py      Score sentences with mBERT and XLM-RoBERTa
enrich_scored.py         Join speaker and conversation metadata for mixed model
analyse_results.R        Main statistical analysis and visualisations
analyse_mixed.R          Log transformation, Q-Q plots, mixed effects model
```

## Data

This project uses the **Bangor Miami Corpus**, a collection of naturalistic
Spanish-English bilingual conversations recorded in Miami (2008-2011).

The corpus is publicly available via TalkBank:
https://talkbank.org/access/BilingBank/Miami.html

Download and extract the corpus before running any scripts. The expected
directory structure after extraction is:

```
miami/
    eng/    (English-dominant conversations, .cha files)
    spa/    (Spanish-dominant conversations, .cha files)
```

The corpus data files are not included in this repository due to TalkBank
terms of use. Please download them directly from TalkBank and cite the
original corpus paper (Deuchar et al., 2014).

## Installation

Python 3.12 is required. Install all Python dependencies with:

```bash
pip install transformers torch sentencepiece sacremoses spacy
python -m spacy download en_core_web_sm
python -m spacy download es_core_news_sm
```

For GPU scoring (strongly recommended), install PyTorch with CUDA support.
Check your CUDA version with `nvidia-smi` and follow the instructions at
https://pytorch.org/get-started/locally/. For CUDA 12.x:

```bash
pip install torch --index-url https://download.pytorch.org/whl/cu121
```

For the R statistical analysis, install R 4.5 or later and the following
packages:

```r
install.packages(c("tidyverse", "lme4", "lmerTest", "ggplot2",
                   "scales", "effsize"))
```

## Running the pipeline

The pipeline consists of five sequential steps. Run them in order from
your working directory.

### Step 1 -- Preprocess the corpus

Parses the raw CHAT files, resolves language tags, enriches POS tags for
embedded-language tokens using spaCy, and outputs a CSV of code-switched
utterances.

```bash
python preprocess_miami.py --input miami/ --output miami_preprocessed_cs_only.csv
```

Output: `miami_preprocessed_cs_only.csv`

### Step 2 -- Generate minimal pairs

For each code-switched utterance, generates invalid counterparts by moving the
switch point to EC-violating positions and translating affected tokens using
Helsinki-NLP MarianMT models. The `--limit` flag is useful for testing.

```bash
python build_pairs.py \
    --input miami_preprocessed_cs_only.csv \
    --output pairs.csv

# Test run with 50 sentences:
python build_pairs.py \
    --input miami_preprocessed_cs_only.csv \
    --output pairs.csv \
    --limit 50
```

Output: `pairs.csv` (4,358 rows: 1,496 valid + 2,862 invalid)

### Step 3 -- Score pseudo-perplexity

Scores all sentences using mBERT and XLM-RoBERTa under both the original
Salazar et al. (2020) method and the corrected Kauf and Ivanova (2023) method.
Run once per method.

```bash
# Salazar et al. method (original, mask one subword at a time):
python score_perplexity.py \
    --input pairs.csv \
    --output pairs_scored_salazar.csv \
    --method salazar

# Kauf & Ivanova method (corrected, mask whole word at a time):
python score_perplexity.py \
    --input pairs.csv \
    --output pairs_scored_kauf.csv \
    --method kauf
```

Each run takes approximately 17 minutes per model on an NVIDIA RTX 3060 Ti.
Output: `pairs_scored_salazar.csv`, `pairs_scored_kauf.csv`

### Step 4 -- Enrich with metadata

Joins speaker and conversation identifiers from the preprocessed corpus into
the scored files, required for the mixed effects model.

```bash
python enrich_scored.py \
    --input pairs_scored_salazar.csv \
    --output pairs_scored_salazar_enriched.csv

python enrich_scored.py \
    --input pairs_scored_kauf.csv \
    --output pairs_scored_kauf_enriched.csv
```

### Step 5 -- Statistical analysis in R

Open RStudio, set the working directory to your project folder, and run the
analysis scripts. Update `INPUT_FILE` and `OUTPUT_DIR` at the top of each
script to switch between the Salazar and Kauf datasets.

For the main results (box plots, accuracy charts, pair-level statistics):

```r
# In analyse_results.R, set:
INPUT_FILE <- "pairs_scored_salazar.csv"
OUTPUT_DIR <- "results_salazar"
source("analyse_results.R")
```

For the mixed effects model and normality checks:

```r
# In analyse_mixed.R, set:
INPUT_FILE <- "pairs_scored_salazar_enriched.csv"
OUTPUT_DIR <- "results_mixed_salazar"
source("analyse_mixed.R")
```

Repeat for the Kauf files to produce the robustness check results.

## Results summary

Under the Salazar et al. scoring method:

| Model        | Pair accuracy | LME beta | p-value  |
|--------------|--------------|----------|----------|
| mBERT        | 70.2%        | 0.14     | .003     |
| XLM-RoBERTa  | 76.7%        | 0.42     | < .001   |

Results replicate under the Kauf and Ivanova correction (69.4% and 78.9%
respectively), confirming that the findings are robust to the choice of
scoring method.
